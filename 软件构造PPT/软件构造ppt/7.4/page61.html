7.4 Debugging
Diagnosis stratagem 5: Learn from others
▪ Sometimes the bug will relate to a widely used technology (your
compiler, for example, or a library or framework you’re using) in
which case there’s a chance that someone else has run afoul of the
same problem before you.
▪ In such instances, a little research on the Web can play dividends.
Perhaps someone has asked a question about the same kind of
failure module on a forum or has written a blog post describing the
pitfall they fell into, which turns out to be exactly the one you find
yourself in.
▪ Google it！

3.4 Object-Oriented Programming (OOP)
How to solve it? --- Defensive copying
▪ We can patch this kind of rep exposure by using defensive copying:
making a copy of a mutable object to avoid leaking out references to
the rep.
▪ Defensive copying is an approach of defensive programming
– Assume clients will try to destroy invariants --- May actually be true
(malicious hackers), but more likely, honest mistakes
– Ensure class invariants survive any inputs, to minimize mutability

3.4 Object-Oriented Programming (OOP)
Invoking overloaded methods
class Animal {
public void eat() {}
}
public class TestUseAnimals {
public static void main (String [] args) {
UseAnimals ua = new UseAnimals();
class Horse extends Animal {
public void eat(String food) {}
}
Animal animalobj = new Animal();
Horse horseobj = new Horse();
public class UseAnimals {
public void doStuff(Animal a) {
System.out.println("Animal");
}
public void doStuff(Horse h) {
System.out.println("Horse");
}
Animal animalRefToHorse = new Horse();
ua.doStuff(animalobj);
Animal
ua.doStuff(horseobj);
Horse
ua.doStuff(animalRefToHorse); Animal
}
}
}
Which overridden version of the method to call is
decided at runtime based on object type, but which
overloaded version of the method to call is based on the
reference type of the argument passed at compile time.

Top-down integration
7.5 Testing and Test-First Programming
▪ The integration process of Top-down integration is performed in a
series of five steps:
– 1. The main control module is used as a test driver and stubs are
substituted for all components directly subordinate to the main control
module.
– 2. Depending on the integration approach selected (i.e., depth or breadth
first), subordinate stubs are replaced one at a time with actual
components.
– 3. Tests are conducted as each component is integrated.
– 4. On completion of each set of tests, another stub is replaced with the real
component.
– 5. Regression testing (discussed later in this section) may be conducted to
ensure that new errors have not been introduced.
– The process continues from step 2 until the entire program structure is
built.

7.1 Robustness & Correctness
Principles of robust programming
▪ Dangerous implements - Users should not gain access to libraries,
data structures, or pointers to data structures.
– This information should be hidden from the user so that the user doesn't
accidentally modify them and introduce a bug in the code.
– When such interfaces are correctly built, users use them without finding
loopholes to modify the interface.
– The interface should already be correctly implemented, so the user does
not need to make modifications. The user therefore focuses solely on his
or her own code.
▪ Can‘t happen - Very often, code is modified and may introduce a
possibility that an “impossible” case occurs. 避免发生小概率事件
– Impossible cases are therefore assumed to be highly unlikely instead.
– The developer thinks about how to handle the case that is highly
unlikely, and implements the handling accordingly.

Example 1: switch statements
9.1 Code Smells  Refactoring
▪ switch statements are very rare in properly designed object-
oriented code
– Therefore, a switch statement is a simple and easily detected “bad smell”
– Of course, not all uses of switch are bad
– A switch statement should not be used to distinguish between various
kinds of object
▪ There are several well-defined refactorings for this case
– Replace conditional with polymorphism
– Motivation: You have a conditional that chooses different behavior
depending on the type of an object.
– Technique: Move each leg of the conditional to an overriding method in a
subclass. Make the original method abstract.

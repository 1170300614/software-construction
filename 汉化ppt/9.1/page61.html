The purposes of code review
9.1 Code Smells  Refactoring
▪ Improving the code
– Finding bugs, anticipating possible bugs, checking the clarity of the code,
and checking for consistency with the project’s style standards.
– At Google, you can’t push any code into the main repository until another
engineer has signed off on it in a code review.
▪ Improving the programmer
– Code review is an important way that programmers learn and teach each
other, about new language features, changes in the design of the project or
its coding standards, and new techniques. In open source projects,
particularly, much conversation happens in the context of code reviews.

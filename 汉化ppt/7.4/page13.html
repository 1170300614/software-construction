Description of Stages
7.4 Debugging
▪ Deferred: The bug, changed to deferred state means the bug is
expected to be fixed in next releases. The reasons for changing the bug
to this state have many factors. Some of them are priority of the bug
may be low, lack of time for the release or the bug may not have major
effect on the software. 被推迟到下个版本修正，优先级低
▪ Duplicate: If the bug is repeated twice or the two bugs mention the
same concept of the bug, then one bug status is changed to
“DUPLICATE”. 重复出现
▪ Pending Reject: If the developers think that a particular behavior of
the system, which the tester reports as a bug has to be same and the
bug is invalid, in that case, the bug is rejected and marked as ‘Pending
Reject’. 无效bug，标记为待拒绝
▪ Rejected/Invalid: Some times developer or team lead can mark the
bug as Rejected or invalid if the system is working according to
specifications and bug is just due to some misinterpretation. 被拒绝

An example for controlling inputs
7.4 Debugging
▪ Debugging will be easier if you first work on reducing the size of
the buggy input to something manageable that still exhibits the
same (or very similar) bug: 保持复现bug的前提下，降低输入规模
– Does the first half of Shakespeare show the same bug? (Binary search!
Always a good technique) 二分法
– Does a single play have the same bug? 某个剧可复现bug
– Does a single speech have the same bug? 某个会话可复现bug
▪ Once you’ve found a small test case, find and fix the bug using that
smaller test case, and then go back to the original buggy input and
confirm that you fixed the same bug.

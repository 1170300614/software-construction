9.2 Refactoring Techniques and Tool Support
Extract Subclass
A class has features that are used
only in some instances.
Create a subclass for that subset of
▪ When a class has features (attributes features.
and methods) that would only be useful in specialized instances,
we can create a specialization of that class and give it those features.
▪ This makes the original class less specialized (i.e., more abstract),
and good design is about binding to abstractions wherever
possible.
public class Person {
private String name;
private String jobTitle;
}
public class Person {
protected String name;
}
public class Employee extends Person {
private String jobTitle;
}

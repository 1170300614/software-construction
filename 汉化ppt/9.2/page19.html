9.2 Refactoring Techniques and Tool Support
Don’t Use Global Variables
▪ Avoid global variables.
▪ A global variable is:
– a variable , a name whose meaning can be changed
– that is global , accessible and changeable from anywhere in the program.
▪ In Java, a global variable is declared public static. The public
modifier makes it accessible anywhere, and static means there is
a single instance of the variable.
▪ In general, change global variables into parameters and return
values, or put them inside objects that you’re calling methods on.
